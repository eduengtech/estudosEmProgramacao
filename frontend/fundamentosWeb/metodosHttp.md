## Métodos HTTP

O **protocolo HTTP** define um conjuntos de**métodos de requisições**, onde cada um realizam um determinado tipo de ação, também conhecido como **HTTP Verbs .** Vamos falar um pouco sobre cada um deles abaixo

### Método GET

Basicamente o método **GET** fazem solicitações de um recurso especifico.Requisições desse tipo deve retornar apenas os dados. 

Fazendo uma analogia usando ainda o restaurante, o verbo **GET** como fazer uma consulta ao cardápio do restaurante, onde retorna as informações, sem alterar nada no sistema. Assim como olhar o cardapio nao altera pedidos, o método GET não deve modificar dados no servidor.

### Método POST
Sempre que queremos enviar uma informação para o servidor para que ela
seja armazenada, o método **POST** é utilizado. Não é à toa que, ao fazer
uma publicação em uma rede social, dizemos que "postamos" uma foto ou um conteúdo.

Na analogia do restaurante, o método **POST** é como fazer um novo pedido:
você envia as informações e o restaurante cria algo novo para você.

### Método PUT

O método PUT é utilizado para atualizar um recurso existente no servidor,
enviando uma nova representação completa desse recurso.

Usando a analogia do restaurante, o PUT é como pegar um pedido já feito
e pedir para a cozinha preparar um novo prato, substituindo totalmente
o pedido anterior.

### Método delete

O método **DELETE** é utilizado para remover um recurso do servidor.

Fazendo uma analogia com um restaurante, é como quando o cliente pede
para o restaurante cancelar um pedido que já foi feito.
